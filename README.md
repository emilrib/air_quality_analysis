# Air Quality Analysis


## Project purpose

This project supports an air quality data pipeline focused on collecting, preparing, and storing OpenAQ air quality data on the pollutant PM2.5 for analysis.

The goal is to build a structured workflow that can ingest air quality data from public APIs, transform it into analysis-ready datasets, and prepare storage structures for downstream reporting, querying, or data warehouse/data lake use cases.
The data contains data from the last 12 months and the pipelines load daily data continuously using a schedule.  

The project is organised around the main stages of a data pipeline:

1. **Ingestion** — collecting raw data from external sources.
2. **Transformation** — cleaning, reshaping, enriching, and preparing data.
3. **Storage preparation** — creating database tables or target structures for processed data.

## Folder details

### `Ingestion/`

This folder is for ingestion-related work.

It is where lambda scripts used to retrieve air quality data from the APIs can be found. In this project, the ingestion layer represents the first stage of the pipeline, before the data is transformed or loaded into a database.

### `Transformation/`

This folder contains the main processing and database preparation scripts.

Files currently found in this folder:

- `OpenAQ Locations Pipeline.py`  
  Used for handling OpenAQ location-related data.

- `OpenAQ Measurements Pipeline .py`  
  Used for handling OpenAQ measurement-related data.

- `RDS Table Creation.py`  
  Used for creating or preparing relational database tables for storing processed data.

### `.venv/`

This folder contains the local Python virtual environment.

It includes installed packages and environment-specific files used to run the project locally. This folder is generated by Python tooling and should not be edited manually.

### `README.md`

This file provides documentation for the repository, including the project purpose, folder structure, and notes about where different parts of the project can be found.

## Notes

- Data collection or landing files belong in `Ingestion/`.
- Data processing and database preparation files belong in `Transformation/`.
- The `.venv/` folder is for local environment management only.
